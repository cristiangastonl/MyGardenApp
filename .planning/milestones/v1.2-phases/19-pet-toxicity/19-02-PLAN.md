**Methodology (per RESEARCH.md В§"Classification Workflow"):**

For each entry (118 total), in batches of ~30 entries per commit (= 4 commits total to keep atomic-commit-friendly):

1. **Look up scientific name on ASPCA's plant list:**
   - Primary URL: `https://www.aspca.org/pet-care/animal-poison-control/toxic-and-non-toxic-plants`
   - Per-species detail pages: append `/cats-plant-list/<slug>` or `/dogs-plant-list/<slug>`.
   - If the entry's `scientificName` (e.g., "Epipremnum aureum") matches an ASPCA entry directly в†’ use that.
   - If a sister-species or genus-only match is in ASPCA в†’ inherit classification per Pitfall 6 rule, cite the related ASPCA entry, note "Genus-inheritance from <related>" in `notes`.
   - If neither species nor genus is in ASPCA в†’ mark `'unknown'` for both species, leave `*_source` empty, set `notes` to `"LATAM-specific вЂ” not in ASPCA"` or `"Not in ASPCA вЂ” no classification possible"`.

2. **Record per-species level:**
   - ASPCA "Toxic to Cats" в†’ `cats_level=toxic`
   - ASPCA "Non-Toxic to Cats" в†’ `cats_level=safe`
   - ASPCA describes mild GI signs without systemic toxicity в†’ `cats_level=caution`
   - Not listed for cats в†’ `cats_level=unknown`
   - Same logic for `dogs_level`.

3. **Record source URL:**
   - `cats_source` = full URL to the cats-specific ASPCA page (or unified per-plant page if newer)
   - `dogs_source` = full URL to the dogs-specific ASPCA page (often the SAME URL вЂ” many plants are listed once with toxicity statements per species)
   - For `'unknown'` entries: leave source empty.

4. **Record symptoms (only for `'caution'` or `'toxic'`):**
   - Format: semicolon-separated EN strings extracted verbatim from ASPCA "Clinical Signs" field.
     - Example: `cats_symptoms="Oral irritation; Intense burning of mouth, lips, tongue; Excessive drooling; Vomiting; Difficulty swallowing"`
   - For `'safe'` and `'unknown'` entries: leave symptoms empty.
   - Cap at 5 symptoms per species (most-clinically-relevant; ASPCA usually lists 3-7).

5. **CSV escaping:**
   - Wrap fields containing commas, semicolons, or quotes in double-quotes per RFC 4180.
   - Escape internal double-quotes by doubling: `""`.
   - Example row: `potus,Epipremnum aureum,interior,toxic,https://www.aspca.org/pet-care/animal-poison-control/cats-plant-list/pothos,toxic,https://www.aspca.org/pet-care/animal-poison-control/dogs-plant-list/pothos,"Oral irritation; Drooling; Vomiting","Oral irritation; Drooling; Vomiting",`
