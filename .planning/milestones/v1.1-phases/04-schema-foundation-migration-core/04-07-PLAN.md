<objective>
Wire the App-level concerns of Phase 4: (1) trigger post-migration **morning-reminder** reschedule when `migrationJustHappened` flips true; (2) render the failure banner above MainTabs when `migrationFailed` is true. This plan is intentionally THIN — both pieces of state and both components were built upstream; this plan only connects them at the App.tsx provider tree.

**B1 fix — sun notifications are NOT in the App-level reschedule.** `scheduleSmartSunNotifications(plants, null)` short-circuits on `!weather` (`notificationScheduler.ts:795-799`) — it would be a no-op at App-level where we have no weather data. Sun notifications are scheduled by TodayScreen's existing `useNotifications` hook once weather loads (typically 1-3 seconds after the user opens the Hoy tab). That existing flow continues to work correctly post-migration because Plan 04 made the scheduler `lightLevel`-aware. The App-level effect handles ONLY the morning reminder, which is weather-independent.

Purpose: Sever storage-layer purity (useStorage stays free of side effects per CONTEXT.md Open Question 2) by lifting the morning-reminder reschedule trigger to App.tsx. Place the banner above MainTabs (CONTEXT.md Open Question 3) so it shows regardless of which tab the user is on.

Output: App.tsx wires both flags; reschedule effect uses Plan 04 lightLevel-aware morning-reminder scheduler without throwing; banner renders conditionally with theme-compliant placement.
</objective>

    ```typescript
    // Post-migration morning-reminder reschedule (SCHEMA-06; B1).
    // Triggered when useStorage flips migrationJustHappened=true after the v0→v1 migration.
    // Lifted to App-level (not inside useStorage) to keep the storage layer pure of side effects.
    //
    // SCOPE LIMIT: morning reminder ONLY. Sun notifications are weather-gated —
    // scheduleSmartSunNotifications(plants, null) short-circuits on !weather (notificationScheduler.ts:795-799).
    // Sun reminders are scheduled by TodayScreen's existing useNotifications hook once weather loads.
    useEffect(() => {
      if (storageLoading) return;
      if (!migrationJustHappened) return;

      let cancelled = false;
      const reschedule = async () => {
        try {
          await cancelAllNotifications();
          const morningTime = notificationSettings?.morningTime ?? '08:00';
          await scheduleMorningReminder(morningTime, plants, null, []);
          // NOTE (B1): scheduleSmartSunNotifications intentionally NOT called here.
          // It requires weather data and would be a no-op at App-level. TodayScreen's
          // existing useNotifications hook handles it once weather loads.
        } catch (rescheduleError) {
          trackEvent('migration_failed', {
            error: String(rescheduleError),
            stage: 'reschedule',
            plantCount: plants.length,
          });
          // Partial reschedule acceptable per CONTEXT.md — do not surface banner from this path.
        } finally {
          if (!cancelled) {
            // Always clear the flag — one-shot reschedule, no retry loop
            acknowledgeMigrationReschedule();
          }
        }
      };
      reschedule();

      return () => { cancelled = true; };
    }, [storageLoading, migrationJustHappened, notificationSettings, plants, acknowledgeMigrationReschedule]);
    ```
