<objective>
Mechanically apply the same Phase 4 mappers to every entry in `PLANT_DATABASE` so future v1.1 reads (Phase 5 health, Phase 6 UI) find a consistent precision-care shape regardless of whether they look up a user plant or a catalog entry. This is the LIGHT-03 / WATER-04 catalog-side application — the user-data side ships in Plans 02-03.

Purpose: One source of truth (Plan 02 mappers) governs both user data and catalog content. Phase 8 catalog rebalance will REPLACE these auto-mapped values with expert-vetted overrides per entry; this plan only seeds them.

Output: PlantDBEntry type extended; codemod script that runs the same `sunHoursToLightLevel` / `applyColdFactor` / `inferWaterMode` mappers against every entry; updated plantDatabase.ts with new fields on every entry; tsc green; grep guard green.
</objective>
