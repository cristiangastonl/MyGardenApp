<objective>
Wire `lightLevel` end-to-end through the plant-identification flow so that:

1. The `IdentifiedPlant` type has an optional `lightLevel` field.
2. The PlantNet client converter (`convertPlantNetResult`) populates `lightLevel` via a defensive ladder (catalog match → derived from sunHours → safe default `bright_indirect`).
3. `IdentificationResults` renders the `LightLevelPicker` for the selected plant, pre-selected with the converter's value, and lets the user override before saving.
4. The saved Plant carries the user-picked `lightLevel`, NOT the auto-derived one (LIGHT-05 requirement: "user can adjust").

Purpose: LIGHT-05 says "Plant identification flow returns plants with lightLevel populated (default bright_indirect if PlantNet doesn't supply enough info, user can adjust before saving)." Phase 7 RESEARCH §Pattern 6 + Pitfall 7 specify the client-side route (no edge-function changes needed for identify-plant in Phase 7 — RESEARCH §Pitfall 7's "preferred Option A" is deferred until catalog rebalance ships in Phase 8 with proper PlantNet sunlight tagging; for now, client-side derivation is the locked path).
