Update `App.tsx` (`AppContentMVP` function):

1. Add imports at top:
   ```typescript
   import * as Notifications from 'expo-notifications';
   import { NavigationContainerRef } from '@react-navigation/native';
   ```

2. Add state and refs inside AppContentMVP:
   ```typescript
   const navigationRef = useRef<NavigationContainerRef<any>>(null);
   const [navReady, setNavReady] = useState(false);
   const [pendingNotificationPlantId, setPendingNotificationPlantId] = useState<string | null>(null);
   ```

3. Add notification response handler function inside AppContentMVP:
   ```typescript
   const handleNotificationResponse = useCallback((response: Notifications.NotificationResponse) => {
     const data = response.notification.request.content.data;
     if (data?.type === 'followup-reminder' && data?.plantId) {
       // Navigate to Hoy tab and signal plant detail open
       navigationRef.current?.navigate('Hoy' as never);
       setPendingNotificationPlantId(data.plantId as string);
     }
   }, []);
   ```

4. Add useEffect for notification listeners, gated on `!storageLoading && navReady`:
   ```typescript
   useEffect(() => {
     if (storageLoading || !navReady) return;

     // Cold-start: check if app was opened via notification tap
     Notifications.getLastNotificationResponseAsync().then((response) => {
       if (response) handleNotificationResponse(response);
     });

     // Foreground/background: listen for notification taps
     const sub = Notifications.addNotificationResponseReceivedListener(handleNotificationResponse);
     return () => sub.remove();
   }, [storageLoading, navReady, handleNotificationResponse]);
   ```

5. Update NavigationContainer to use ref and onReady:
   ```typescript
   <NavigationContainer ref={navigationRef} onReady={() => setNavReady(true)}>
   ```

6. Pass `pendingNotificationPlantId` and a clear function to MainTabs. Since MainTabs renders TodayScreen via Tab.Navigator, use React Context or pass via screenProps. SIMPLEST approach: Create a minimal React context in App.tsx:
   ```typescript
   export const NotificationContext = React.createContext<{
     pendingPlantId: string | null;
     clearPendingPlantId: () => void;
   }>({ pendingPlantId: null, clearPendingPlantId: () => {} });
   ```
   Wrap the NavigationContainer with `<NotificationContext.Provider value={{ pendingPlantId: pendingNotificationPlantId, clearPendingPlantId: () => setPendingNotificationPlantId(null) }}>`.

Update `src/screens/TodayScreen.tsx`:

1. Import `NotificationContext` from `../../App` (or wherever exported -- adjust path based on actual project structure; likely `'../../App'` from screens folder).

2. Consume context:

   const { pendingPlantId, clearPendingPlantId } = useContext(NotificationContext);

3. Add useEffect to auto-open plant detail when pendingPlantId is set:
   ```typescript
   useEffect(() => {
     if (pendingPlantId && plants.length > 0) {
       const plant = plants.find(p => p.id === pendingPlantId);
       if (plant) {
         setDetailPlant(plant);
       }
       clearPendingPlantId();
     }
   }, [pendingPlantId, plants]);
   ```

This ensures: cold-start waits for storage+nav ready, then navigates to Hoy and signals TodayScreen to open the plant detail modal.

IMPORTANT TIMING GUARD: The `useEffect` in App.tsx only runs when `!storageLoading && navReady`. This prevents the "Navigator is not mounted" crash and the "plant not loaded yet" race condition documented in STATE.md and RESEARCH.md pitfall 1.
