### Copywriting rules for this phase

- Empty state: not applicable (camera is always available in an active chat session)
- Error state: permission denial is the only error path — handled by the inline message above
- Destructive actions: none in this phase (pending-photo remove uses existing ✕ button — no new confirmation needed)
- Primary CTA: the photo button has no text label — it is the 📷 emoji at 20px on a 44px touch target

## Interaction States

### Photo button tap flow

```
Tap 📷 (canSendChat=true)
  └── iOS: ActionSheetIOS slides up (native)
  └── Android: custom Modal slides up from bottom
        ├── User taps "Take a photo"
        │     ├── Permission granted → camera opens → photo taken → thumbnail appears in pending area
        │     └── Permission denied → modal closes, cameraPermissionDenied=true → inline info box appears
        ├── User taps "Choose from gallery" → existing gallery flow (unchanged)
        └── User taps "Cancel" / back-press → resolve(null), dismiss, no state change

Tap 📷 (canSendChat=false, free-tier limit reached)
  └── Paywall screen appears (existing behavior — mirrors chat input disabled state)
```

### Pending photo thumbnail (existing, no changes)

- 80x80px, `borderRadius.md` (10px)
- Remove button: 24x24px circle, `colors.dangerBg`, ✕ at 12px `DMSans_600SemiBold` `colors.dangerText`
- Positioned above the input row

### In-message photo (existing, no changes)

- 160x160px, `borderRadius.md` (10px), inside the message bubble

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Photo button touch target | 44x44px minimum (matches `MIN_TOUCH_SIZE` from theme.ts) |
| Action sheet cancel | Always at index 0 on iOS (native behavior); explicit Cancel row on Android |
| Android back-press | `onRequestClose` resolves Promise with `null` — no navigation trap |
| Permission-denial link | "Open Settings" tappable area minimum 44px height via `hitSlop` `{ top: 12, bottom: 12, left: 12, right: 12 }` (existing `hitSlop` from theme.ts) |
